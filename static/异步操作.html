异步操作

异步操作的模式

回调函数

回调函数是异步操作最基本的方法。

下面是两个函数f1和f2，编程的意图是f2必须等到f1执行完成，才能执行。

function f1() {
  // ...
}

function f2() {
  // ...
}

f1();
f2();

但如果f1是异步函数，则f2会在f1之前就执行

我们可以把f2写成f1的回调函数

function f1(callback) {
  // ...
  callback();
}

function f2() {
  // ...
}

f1(f2);
回调函数的优点是实现简单，容易理解 缺点是不易于代码的阅读和维护，会使代码高度耦合，程序结构混乱，流程难以追踪（尤其是多个回调函数嵌套的情况），而且每个任务只能指定一个回调函数

事件监听

这种方式采用事件驱动模式，异步任务的执行并不取决于代码的顺序，而是取决于任务监听的事件是否发生

我们可以给f1()绑定一个事件，当这个事件发生时执行f2

在f1执行完后出发这个事件来让f2()执行

这种方式的优点是容易理解，能够去耦合化，能够绑定多个事件，且每个事件可以指定多个回调函数，有利于实现代码模块化 缺点是整个程序都要变成事件驱动型，运行流程会变得很不清晰

发布/订阅

我们可以把事件理解为“信号”，当f1()执行完毕，就向任务中心发布一个信号，f2()可以向任务中心来订阅这个信号，当检测到信号已经发布时就开始执行任务，完成任务后就可以取消订阅，这种方式就被称为“发布/订阅模式”

这种方法的性质与“事件监听”类似，但是明显优于后者。因为可以通过查看“消息中心”，了解存在多少信号、每个信号有多少订阅者，从而监控程序的运行。

异步操作的控制流程

如果有多个异步操作，就存在一个流程控制的问题：如何确定异步操作执行的顺序，以及如何保证遵守这种顺序。

串行执行

编写一个流程控制函数，当一个任务完成才执行下一个

优点：占用资源少

缺点：运行很慢

并行执行

所有任务同时进行

优点：运行速度快

缺点：会耗尽系统资源，拖慢运行速度

串行与并行结合

在并行执行的同时设置一个条件，限制最大可以同时运行的任务，当前一组任务完成才进行下一组任务，避免过分占用系统资源

效率和资源最佳平衡
